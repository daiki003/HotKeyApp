# HotKeyApp Project Rules

このファイルは HotKeyApp リポジトリで作業するエージェント向けのプロジェクト固有ルールです。

## UI ルール

- 設定画面などで「特定の値 2 つを 1 組で登録する」UIは、`Views/GitSettingsDialog.xaml` の Git 設定エイリアス画面と同じパターンに統一する。
- 標準パターンは次の構成とする。
  - 左端にドラッグハンドル `☰`
  - 2 つの入力欄
  - 右端にインラインの `削除` ボタン
  - 下部に `新規追加` ボタン
- 操作感もそろえる。
  - ドラッグ＆ドロップで並び替え可能
  - `Shift+Up` / `Shift+Down` で行の並び替え可能
  - 矢印キーで隣接セルや前後行へフォーカス移動できる
- 今後 `定数定義`、`選択式テンプレート`、その他のキーと値のペア登録画面を追加する場合も、この標準パターンを優先する。

## 実装方針

- 2 項目ペア登録 UI を新設するときは、既存の `SettingsWindow` 内で個別実装を増やす前に、Git 設定エイリアス画面との差分が最小になるように見た目とイベント構成を揃える。
- ラベル名だけが違う画面では、列構成、余白、ボタンスタイル、入力欄スタイルをできるだけ共通化する。
